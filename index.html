Conectar con Supabase

Ingresa las credenciales de tu proyecto Supabase. Solo se hace una vez — se guardan localmente.

📋 ¿Cómo obtener estas credenciales?
1. Ve a supabase.com → Nuevo proyecto
2. En tu proyecto → Settings → API
3. Copia Project URL y anon public key
📐 SQL para crear las tablas (ejecutar en Supabase → SQL Editor):
-- 1. Usuarios CREATE TABLE usuarios ( id BIGSERIAL PRIMARY KEY, nombre TEXT NOT NULL, usuario TEXT UNIQUE NOT NULL, password TEXT NOT NULL, rol TEXT DEFAULT 'cobrador', color TEXT DEFAULT '#1565C0', activo BOOLEAN DEFAULT true, created_at TIMESTAMPTZ DEFAULT NOW() ); -- 2. Clientes CREATE TABLE clientes ( id BIGSERIAL PRIMARY KEY, nombre TEXT NOT NULL, telefono TEXT, direccion TEXT, email TEXT, color TEXT DEFAULT '#1565C0', activo BOOLEAN DEFAULT true, created_at TIMESTAMPTZ DEFAULT NOW() ); -- 3. Préstamos CREATE TABLE prestamos ( id BIGSERIAL PRIMARY KEY, cliente_id BIGINT REFERENCES clientes(id), monto NUMERIC NOT NULL, interes NUMERIC DEFAULT 10, plazo INT DEFAULT 6, fecha_inicio DATE, estado TEXT DEFAULT 'activo', cobrador_id BIGINT REFERENCES usuarios(id), notas TEXT, created_at TIMESTAMPTZ DEFAULT NOW() ); -- 4. Pagos CREATE TABLE pagos ( id BIGSERIAL PRIMARY KEY, prestamo_id BIGINT REFERENCES prestamos(id), monto NUMERIC NOT NULL, fecha DATE NOT NULL, metodo TEXT DEFAULT 'efectivo', notas TEXT, created_at TIMESTAMPTZ DEFAULT NOW() ); -- Usuario admin por defecto INSERT INTO usuarios (nombre, usuario, password, rol, color) VALUES ('Administrador', 'admin', '1234', 'admin', '#1565C0'); -- Deshabilitar RLS para uso simple ALTER TABLE usuarios DISABLE ROW LEVEL SECURITY; ALTER TABLE clientes DISABLE ROW LEVEL SECURITY; ALTER TABLE prestamos DISABLE ROW LEVEL SECURITY; ALTER TABLE pagos DISABLE ROW LEVEL SECURITY;
